### User Stories
1. As a system administrator, I want the /health endpoint to return the service status so that I can monitor the health of the application.
2. As a developer, I need the endpoint to include uptime metrics so that I can track the service availability.
3. As a reliability engineer, I require dependency checks in the response to ensure all critical services are operational.
4. As a DevOps engineer, I need the endpoint to return errors gracefully with appropriate status codes for easier debugging.
5. As a stakeholder, I want the response time to be under 500ms to ensure the endpoint is lightweight and performant.
