Why AI Crew?

AI Sovereignty, centralised standards, and enterprise control

AI Sovereignty

Run self-hosted models on your own OpenShift cluster. No data leaves the organisation. Full control over intellectual property, compliance, and model selection — zero dependency on external LLM cloud providers.

Remote AI Agents on OCP

Agents run as containers on OpenShift, not on developer laptops. Horizontally scalable, air-gapped capable, and deployed via Helm — a true production-grade AI platform.

Centralised Coding Standards

26+ externalized prompt files encode your organisation’s engineering practices — architecture patterns, naming conventions, testing strategies — applied consistently across every project the agents build.

Cost Management & Observability

Budget tracking with per-project cost limits and circuit breakers. Per-agent model selection (Manager / Worker / Reviewer) optimises cost vs. quality. Full progress reporting and job-level observability across all builds.

How It Fits Together

1. Prompts Feed the Loop

26+ externalized prompt files. Meta Agent enriches them per project. Injected at each phase transition.

2. State Machine Drives Phases

Sequential transitions with rollback gates. Each phase completes before the next starts.

3. Task Manager Tracks Work

SQLite-backed registry with per-file granularity. Resume after failure, incremental builds.

4. Error Recovery & Retry

Automatic retry with enriched context. Code validator catches stubs, syntax errors, and import issues.

Refactoring, Refinement & Prompts

Code Refactoring

Migrate legacy codebases to a new tech stack using Architect and Executor agents.

  • Analyse codebase → generate blueprint → apply file-by-file
  • Preserves business logic while modernising
Use case: Spring Boot 2 → 3, monolith → microservices

Refinement

Apply natural-language edits to existing codebases — describe changes in plain English.

  • File-level or project-wide scope, context-aware, iterative
  • Available via Web UI, VS Code, and Jira
Example: “Add pagination to the /users endpoint”
