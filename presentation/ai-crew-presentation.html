Motivation

Why was this built?

Enterprise-Scale Coding Agents

AI coding agents can go far beyond IDE-level copilots. This project demonstrates orchestrating full software delivery pipelines — Product Owner, Designer, Architect, Developer, Frontend, DevOps — at enterprise scale.

☁

Showcasing Red Hat OpenShift AI as the platform for hosting and running AI agents — leveraging OCP-native model serving, UBI containers, and Tekton pipelines for a truly cloud-native AI workflow.

Feature: MTA Migration

Migration Toolkit for Applications (MTA)

Red Hat's MTA analyses Java applications and produces reports listing migration issues. The AI Crew automates the remediation of those issues using specialised agents.

Supported Migrations

javax → jakarta
Java EE to Jakarta EE namespace migration
EAP → Quarkus
JBoss EAP to Quarkus modernisation
Custom Rules
Any MTA ruleset or custom report format

How the AI Crew Automates It

  • Parse MTA reports (issues.json + files.json) into structured issue records
  • Resolve file paths from the report to the actual workspace
  • Apply changes per file using the Migration Execution Agent with 4-tier context
  • Validate structural integrity (package, class name, line count)

MTA Migration: How It Works

Six-Phase Migration Pipeline

0
Pre-migration Git Snapshot
Capture baseline for diff & rollback
1
Parse MTA Report
Read issues.json & files.json → structured records
2
Resolve File Paths
Map report paths to workspace locations
3
AI Agent Applies Changes
Per-issue migration with 4-tier context
4
Deterministic Sweep
javax → jakarta find-and-replace pass
5
Retry Failed Issues
Up to 6 retries with enriched context
6
Post-migration Validation
Structural checks: package, class name, line count
Deterministic + AI

Combines rule-based sweeps with LLM reasoning for complex changes

4-Tier Context

Issue details, file content, migration hints, and project context

Resilient

Git snapshot baseline enables safe rollback at any point

Refactoring & Refinement

Code Refactoring

Migrate legacy codebases to a new tech stack using specialised Architect and Executor agents.

  • Analyses existing codebase structure
  • Generates a refactoring blueprint
  • Applies changes file-by-file
  • Preserves business logic while modernising
Use case: Spring Boot 2 → Spring Boot 3, monolith → microservices

Refinement

Apply natural-language edits to existing codebases — describe what you want changed in plain English.

  • File-level or project-wide scope
  • Context-aware — understands project structure
  • Iterative — refine until satisfied
  • Available via Web UI, VS Code, and Jira
Example: “Add pagination to the /users endpoint”

Automatically deploy generated applications into a live Kubernetes sandbox environment for validation and testing.

  • Spin up ephemeral namespaces on OpenShift / K8s
