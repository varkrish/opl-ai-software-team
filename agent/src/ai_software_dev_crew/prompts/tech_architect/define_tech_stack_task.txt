You MUST enumerate concrete named route files — never a bare `routes/` folder with no files inside it.
Example for a task-management REST API with Apache Camel + Spring Boot:
```text
backend/
├── pom.xml
├── src/
│   ├── main/
│   │   ├── java/com/taskmanagement/
│   │   │   ├── TaskManagementApplication.java   # [SOURCE] Spring Boot entrypoint
│   │   │   ├── config/
│   │   │   │   └── CamelRestConfig.java          # [SOURCE] REST DSL / component config
│   │   │   ├── routes/
│   │   │   │   ├── TaskRoute.java                # [SOURCE] extends RouteBuilder — /tasks CRUD
│   │   │   │   └── UserRoute.java                # [SOURCE] extends RouteBuilder — /users CRUD
│   │   │   ├── processors/
│   │   │   │   └── TaskValidationProcessor.java  # [SOURCE] implements Processor
│   │   │   └── model/
│   │   │       └── Task.java                     # [SOURCE] POJO/DTO exchanged on the route
│   │   └── resources/
│   │       └── application.properties            # [CONFIG]
│   └── test/
│       └── java/com/taskmanagement/routes/
│           └── TaskRouteTest.java                # [SOURCE] CamelTestSupport / AdviceWith
```
